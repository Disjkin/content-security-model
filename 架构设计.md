## 2. 技术选型

| 层次 | 技术 | 选型理由 |
|------|------|----------|
| Web 框架 | FastAPI | 高性能异步框架，自带 OpenAPI 文档 |
| 敏感词匹配 | Trie 树 (前缀树) | O(m) 匹配复杂度，适合大规模词库 |
| 模型推理 | TF-IDF + scikit-learn | 轻量级，模型文件小，可训练更新 |
| 文本处理 | OpenCC + re | 繁简转换 + 正则预处理 |
| 中文分词 | jieba | 成熟稳定的中文分词库 |
| 配置管理 | YAML | 人类可读，易于版本管理 |
| 数据模型 | Pydantic | 与 FastAPI 深度集成，类型安全 |
| 日志 | Python logging | 标准库，轻量无依赖 |
| 测试 | pytest + httpx | 成熟测试框架 + 异步 HTTP 测试 |

### 3.2 Trie 树模块 (`trie.py`)

**职责**: 实现高效的前缀树数据结构，支持多模式字符串匹配。

```
TrieNode
├── children: Dict[str, TrieNode]   # 子节点映射
├── is_end: bool                    # 是否为词尾
└── category: Optional[str]         # 违规分类标记

TrieTree
├── insert(word, category)          # 插入词汇
├── search(text) → List[Match]      # 搜索所有匹配
├── remove(word)                    # 删除词汇
├── build_from_file(path, category) # 从文件批量构建
└── size → int                      # 词库大小
```

### 3.3 文本预处理器 (`preprocessor.py`)

**职责**: 对输入文本进行标准化处理，消除常见的绕过手段。

```
Preprocessor
├── normalize(text) → str           # 主入口：标准化文本
│   ├── unicode_normalize()         # Unicode 归一化 (NFKC)
│   ├── remove_special_chars()      # 去除干扰字符
│   ├── traditional_to_simplified() # 繁体转简体
│   ├── lowercase()                 # 统一小写
│   └── normalize_variants()        # 常见变体转换
└── skip_chars: Set[str]            # 可忽略的干扰字符集
```

### 3.4 敏感词过滤器 (`sensitive_filter.py`)

**职责**: 基于 Trie 树的敏感词匹配引擎。

```
SensitiveFilter
├── __init__(word_manager)          # 初始化，关联词库管理器
├── filter(text) → FilterResult     # 执行敏感词匹配
│   ├── 预处理文本
│   ├── 对各类别 Trie 树匹配
│   └── 返回命中词、位置、类别
├── rebuild_trees()                 # 重建 Trie 树（词库更新后）
└── get_stats() → dict              # 获取词库统计
```

### 3.5 模型推理分类器 (`model_classifier.py`)

**职责**: 基于机器学习的语义违规检测，补充敏感词无法覆盖的隐晦表达。

```
ModelClassifier
├── __init__(config)
│   ├── 加载 TF-IDF 向量化器
│   └── 加载分类模型
├── classify(text) → ClassificationResult
│   ├── 文本预处理 + jieba 分词
│   ├── TF-IDF 向量化
│   ├── 模型推理 → 各类别概率
│   └── 返回分类结果 + 置信度
├── is_available() → bool           # 模型是否已加载
├── train(data_path)                # 训练/更新模型
└── get_labels() → List[str]        # 获取类别标签
```

**模型架构**:
- 向量化：TF-IDF（基于 jieba 分词）
- 分类器：LinearSVC / LogisticRegression
- 标签：`safe` / `politics` / `gambling` / `pornography`
- 模型文件：pickle 序列化，< 50MB

### 3.6 检测引擎 (`detector.py`)

**职责**: 核心检测逻辑，协调敏感词过滤和模型推理，执行组合策略。

```
ContentDetector
├── __init__(config)
│   ├── 初始化预处理器
│   ├── 初始化敏感词过滤器
│   └── 初始化模型分类器（可选）
│
├── detect(text) → DetectionResult
│   ├── Step 1: 文本预处理
│   ├── Step 2: 敏感词匹配（快速层）
│   ├── Step 3: 模型推理（语义层，可配置是否启用）
│   ├── Step 4: 结果聚合（加权融合）
│   ├── Step 5: 风险等级计算
│   └── Step 6: 返回结构化结果
│
├── detect_batch(texts) → List[DetectionResult]
│   └── 批量检测
│
└── reload()
    └── 热加载词库 + 模型
```

**组合策略**:
| 场景 | 策略 |
|------|------|
| 敏感词命中 + 模型违规 | 高置信度违规，风险等级提升 |
| 敏感词命中 + 模型正常 | 低置信度，可能误报 |
| 敏感词未命中 + 模型违规 | 隐晦违规，按模型置信度判定 |
| 敏感词未命中 + 模型正常 | 安全 |

**风险等级计算规则**:
| 等级 | 条件 |
|------|------|
| critical | 双层均判定违规，或涉政类命中 ≥ 2 个词 |
| high | 敏感词命中 ≥ 1 个词，或模型置信度 > 0.8 |
| medium | 仅模型判定违规且置信度 0.5-0.8 |
| low | 未命中敏感词且模型判定正常 |

### 3.7 API 接口 (`api.py`)

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/v1/detect` | 单条文本检测 |
| POST | `/api/v1/detect/batch` | 批量文本检测 |
| GET | `/api/v1/words/{category}` | 查询指定类别词库 |
| POST | `/api/v1/words/{category}` | 添加敏感词 |
| DELETE | `/api/v1/words/{category}` | 删除敏感词 |
| POST | `/api/v1/reload` | 手动触发词库热加载 |
| POST | `/api/v1/model/train` | 触发模型训练 |
| GET | `/api/v1/stats` | 检测统计信息 |
| GET | `/health` | 健康检查 |


```
  │
  ▼

  │
  ▼

  │  - 大小写统一

  ▼                        ▼
SensitiveFilter         ModelClassifier

  │  - Trie 树搜索         │  - jieba 分词

  │
  ▼
```

## 5. 安全设计

### 5.1 API 鉴权
- 使用 API Key 机制
- Key 通过 HTTP Header (`X-API-Key`) 传递
- 支持多 Key 管理，可按 Key 限流

### 5.2 限流策略
- 全局 QPS 限制
- 单 Key QPS 限制
- 批量检测数量上限

### 5.3 数据安全
- 检测文本不持久化（除非开启日志）
- 日志中敏感词脱敏显示
- 词库文件权限控制

## 6. 扩展性设计

### 6.1 新增违规类型
1. 在 `sensitive_words/` 下新增词库文件
2. 在 `config.yaml` 中注册新类别
3. 引擎自动加载，无需修改代码

### 6.2 模型升级路径
```
当前: TF-IDF + LinearSVC (轻量)
  ↓
下一步: 微调中文 BERT 模型 (高精度)
  ↓
未来: 多模态模型 (文本+图片)
```

### 6.3 水平扩展
- 无状态设计，可多实例部署
- 通过负载均衡分发请求
- 词库通过共享存储保持一致
